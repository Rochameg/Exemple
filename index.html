Système de Gestion d’Assurance

Vue d’ensemble

Il s’agit d’un système de gestion d’assurances développé avec Flask, conçu pour gérer les clients, les polices d’assurance et les sinistres. L’application offre une interface web destinée aux agents et administrateurs d’assurance pour gérer efficacement leurs opérations quotidiennes. Le système inclut l’authentification des utilisateurs, la gestion des clients, l’administration des polices et le traitement des sinistres.

Architecture Frontend

  • Moteur de templates : Jinja2 avec Flask (framework web Python)
  • Framework CSS : Bootstrap 5 pour un design réactif
  • Icônes : Font Awesome pour une iconographie cohérente
  • Mise en page : Template de base avec héritage de blocs pour une interface cohérente
  • Formulaires : WTForms pour la validation et le rendu des formulaires

Architecture Backend

  • Framework : Flask (framework web Python)
  • ORM de base de données : SQLAlchemy avec l’extension Flask-SQLAlchemy
  • Authentification : Flask-Login pour la gestion des sessions
  • Sécurité des mots de passe : Outils de hachage de mots de passe de Werkzeug
  • Gestion des formulaires : Flask-WTF pour la protection CSRF et la validation

Stockage des données

  • Base de données : PostgreSQL (base de données relationnelle adaptée à la production)
  • ORM : SQLAlchemy avec un modèle déclaratif de base
  • Gestion de connexion : Recyclage de pool et pré-vérification activés pour plus de fiabilité (Améliorent les performances en évitant de recréer des connexions à chaque requête.)
  • Migrations : Schéma de base géré via les définitions des modèles SQLAlchemy

Composants clés

Modèles (models.py)

  • Modèle Utilisateur : Gère les utilisateurs du système avec authentification
    • Hachage et vérification du mot de passe
    • Gestion des sessions utilisateur
    • Contrôle d’accès administrateur
  • Modèle Client : Gère les informations des clients d’assurance
    • Détails personnels et informations de contact
    • Système de numérotation client unique
    • Relation avec les polices

Formulaires (forms.py)

  • Formulaire de Connexion : Authentification utilisateur avec validation
  • Formulaire d’Inscription : Création de nouveaux comptes
  • Formulaire Client : Gestion des informations client (partiellement implémenté)
  • Validation des formulaires avec gestion complète des erreurs

Routes (routes.py)

  • Routes d’authentification : Connexion, déconnexion, inscription
  • Tableau de bord : Interface principale de l’application
  • Gestion des clients : Opérations CRUD sur les clients
  • Gestion des polices : Administration des polices d’assurance (référencée)
  • Gestion des sinistres : Traitement des sinistres (référencé)

Templates

  • Template de base : Mise en page cohérente avec navigation et style
  • Pages d’authentification : Interfaces de connexion et d’inscription
  • Gestion des clients : Interfaces pour lister, ajouter et modifier des clients
  • Tableau de bord : Vue d’ensemble avec statistiques et actions rapides
  • Templates de polices et sinistres : Interfaces d’administration

Flux de données

  1. Authentification : Les utilisateurs se connectent via Flask-Login()
  2. Traitement des requêtes : Les routes Flask traitent les requêtes HTTP et les soumissions de formulaire
  3. Validation des données : WTForms valide les données avant le traitement
  4. Opérations en base de données : L’ORM SQLAlchemy gère les interactions avec la base
  5. Rendu des réponses : Les templates Jinja2 génèrent les réponses HTML avec les données
  6. Gestion de session : Flask-Login maintient les sessions utilisateur entre les requêtes

Dépendances principales

  • Flask : Framework web principal
  • Flask-SQLAlchemy : Intégration ORM
  • Flask-Login : Gestion des sessions utilisateur
  • Flask-WTF : Gestion des formulaires et protection CSRF
  • psycopg2-binary : Connecteur PostgreSQL
  • email-validator : Validation des emails pour les formulaires

Dépendances frontend (via CDN)

  • Bootstrap 5 : Framework CSS pour un design réactif
  • Font Awesome : Bibliothèque d’icônes pour l’interface utilisateur

Dépendances de développement

  • Jinja2 : Moteur de templates (inclus avec Flask)

Préférences utilisateur

Style de communication préféré : Langage simple et courant.
